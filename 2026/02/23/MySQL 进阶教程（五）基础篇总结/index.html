六、性能优化建议

6.1 索引优化

索引使用原则：

  1. 在 WHERE 和 JOIN 的列上创建索引
  2. 使用联合索引时注意最左前缀原则
  3. 避免在索引列上使用函数
  4. 避免 SELECT *

6.2 查询优化

6.3 表结构优化

  1. 选择合适的数据类型

    • 能用 TINYINT 就不用 INT
    • 金额使用 DECIMAL
    • 字符串长度合理

  2. 范式与反范式

    • 遵循三大范式减少冗余
    • 适当冗余提高查询性能

  3. 分区表

    • 大表按时间或范围分区
    • 提高查询和管理效率

八、常见问题 FAQ

Q1: COUNT(*)、COUNT(1)、COUNT(列) 的区别？

Q2: DELETE、TRUNCATE、DROP 的区别？

命令 类型 作用 可回滚
DELETE DML 删除符合条件的行 是
TRUNCATE DDL 清空整张表 否
DROP DDL 删除整个表 否

Q3: WHERE 和 HAVING 的区别？

  • WHERE：在分组前过滤，不能使用聚合函数
  • HAVING：在分组后过滤，可以使用聚合函数

Q4: INNER JOIN 和 LEFT JOIN 的选择？

  • 只返回匹配记录：INNER JOIN
  • 需要保留左表全部记录：LEFT JOIN

Q5: 如何避免 SQL 注入？

  1. 使用预编译语句（Prepare Statement）
  2. 使用 ORM 框架
  3. 验证用户输入
  4. 最小权限原则
