MySQL 进阶教程 04-05 窗口框架详解

窗口框架（Window Frame）用于精确定义窗口函数操作的数据范围。本文将详细讲解 ROWS 和 RANGE 的区别，以及各种窗口边界的使用方法。

一、窗口框架概述

1.1 什么是窗口框架？

窗口框架 是窗口函数中用于定义”窗口”具体范围的部分，决定了哪些行参与计算。

1.2 默认窗口框架

关键规则：

  • 有 ORDER BY 无框架子句：默认 RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  • 无 ORDER BY：使用整个分区

二、ROWS vs RANGE 的区别

2.1 ROWS - 基于行范围

ROWS 基于物理行位置定义窗口。

2.2 RANGE - 基于值范围

RANGE 基于逻辑值（ORDER BY 列的值）定义窗口。

2.3 ROWS vs RANGE 对比总结

对比项 ROWS RANGE
定义方式 物理行位置 逻辑值范围
相同值处理 每行独立计算 相同值视为一组
性能 通常更快 可能需要额外排序
适用场景 移动平均、累计 值范围分析
MySQL 默认 - RANGE（有 ORDER BY 时）

2.4 实例对比

解释：

  • ROWS：严格按物理行累加
  • RANGE：相同值的行视为一个逻辑组，同时加入窗口

关键字 含义
UNBOUNDED PRECEDING 分区的第一行
UNBOUNDED FOLLOWING 分区的最后一行
CURRENT ROW 当前行
n PRECEDING 当前行之前的第 n 行
n FOLLOWING 当前行之后的第 n 行

八、小结

窗口框架速查表

框架类型 语法 适用场景
ROWS ROWS BETWEEN n PRECEDING AND CURRENT ROW 移动平均、累计
RANGE RANGE BETWEEN n PRECEDING AND CURRENT ROW 值范围分析
GROUPS GROUPS BETWEEN n PRECEDING AND CURRENT ROW 逻辑组分析

常用边界组合

组合 含义 示例
UNBOUNDED PRECEDING AND CURRENT ROW 从起点到当前行 累计求和
CURRENT ROW AND UNBOUNDED FOLLOWING 从当前行到终点 剩余总和
n PRECEDING AND CURRENT ROW 前 n 行到当前行 移动窗口
n PRECEDING AND n FOLLOWING 前后各 n 行 居中窗口
UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING 整个分区 首尾值

核心要点

  1. ROWS 基于物理行，RANGE 基于值范围
  2. 有 ORDER BY 无框架时，默认使用 RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  3. LAST_VALUE 需要显式指定 UNBOUNDED FOLLOWING
  4. 窗口框架影响性能，应根据需求选择最合适的配置
