MySQL 运维篇 04-09 MyCat 一致性 Hash

一致性 Hash 是一种分布式哈希技术，可以减少扩容时的数据迁移量。本文将介绍一致性 Hash 的原理和在 MyCat 中的应用。

一、一致性 Hash 介绍

1.1 什么是一致性 Hash

一致性 Hash：将数据和节点映射到一个哈希环上，数据存储在顺时针方向最近的节点上。

1.3 虚拟节点

虚拟节点：每个物理节点对应多个虚拟节点，使数据分布更均匀。

六、一致性 Hash vs 取模

6.1 对比

特性 一致性 Hash 取模分片
数据分布 均匀 均匀
扩容影响 小 大
实现复杂度 高 低
查询效率 高 高

6.2 选择建议

场景 推荐
需要频繁扩容 一致性 Hash
分片数固定 取模分片
数据量大 一致性 Hash
简单场景 取模分片

七、小结

一致性 Hash 要点

  1. 哈希环：数据和节点映射到环上
  2. 虚拟节点：使数据分布更均匀
  3. 扩容优势：只影响相邻节点
  4. 适用场景：需要频繁扩容的场景
