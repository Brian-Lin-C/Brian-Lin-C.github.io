八、总结

提醒自己：

  1. 只捕获能处理的异常：不要为了捕获而捕获
  2. 明确异常类型：避免使用 bare except
  3. 记录异常信息：方便调试和问题追踪
  4. 及时清理资源：使用 with 或 finally
  5. 合理的异常层次：不要过度嵌套 try-except
  6. 自定义异常要有意义：有助于问题定位

  4. 编写一个配置文件加载器，处理各种可能的错误
