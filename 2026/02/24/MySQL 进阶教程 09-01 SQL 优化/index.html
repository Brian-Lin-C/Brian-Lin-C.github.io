type 说明 优化建议

Using where 使用 WHERE 过滤 正常

Using filesort 使用文件排序 需要优化
NULL 无特殊 正常

十、SQL 优化 Checklist

10.1 SELECT 优化

  • 避免 SELECT *，只查需要的列
  • 使用覆盖索引
  • 避免前缀通配符 LIKE ‘%xxx’
  • 避免在索引列上使用函数

10.2 WHERE 优化

  • WHERE 条件列有索引
  • 避免 OR 条件（使用 UNION）
  • 避免类型隐式转换

10.3 JOIN 优化

  • JOIN 条件列有索引
  • 小表驱动大表
  • 使用 EXISTS 代替 IN（子查询结果集大时）

10.4 索引优化

  • 定期 ANALYZE TABLE
  • 删除未使用的索引
  • 避免冗余索引

十一、小结

优化原则

  1. 先分析后优化：使用 EXPLAIN 分析问题
  2. 先 SQL 后索引：先优化 SQL 写法，再考虑索引
  3. 先局部后整体：先优化最慢的查询
  4. 权衡读写比例：读多写少多建索引，写多读少精简索引

核心技巧速查

COUNT 慢 覆盖索引/缓存
全表扫描 添加索引
文件排序 ORDER BY 列加索引
临时表 GROUP BY 列加索引
