前言

这篇教程是写给我自己的 Markdown 语法备忘录。记录常用的 Markdown 语法和博客配置，方便以后写博客时快速查阅。

提醒自己：写博客不是为了追求完美，而是为了记录和分享。不要等到什么都准备好了才开始，先写起来，在写作中不断完善。

一、Markdown 基础语法

1.1 标题

使用 # 符号来定义标题：

提醒自己：

  • 文章标题用一级标题，正文中从二级标题开始
  • 标题层级要清晰，不要跳级使用（比如不要从 ## 直接跳到 ####）
  • 每个大章节用 ##，子内容用 ###

1.2 段落与换行

提醒自己：

  • 段落不要太长，3-5 句一段比较合适
  • 代码教程中，解释和代码示例之间要空行，方便阅读

1.3 文本样式

样式 语法 示例
粗体 **文本** 重要内容
斜体 *文本* 强调一下
删除线 ~~文本~~ 过时的内容

提醒自己：

  • 粗体用于强调关键概念和重要提示
  • 不要滥用粗体，满篇都是重点就没有重点了
  • 斜体可以用来标注英文术语或书名

1.4 引用块

提醒自己：引用块适合用于：

  • 引用官方文档的原文
  • 标注注意事项
  • 记录他人的观点

  • 有顺序的步骤用有序列表（比如教程的操作步骤）
  • 任务列表可以用来做 TODO 清单

三、代码展示

3.1 行内代码

效果：使用 console.log() 输出，使用 print() 打印。

3.2 代码块

支持的语言：javascript、python、sql、java、cpp、html、css、json、yaml 等。

提醒自己：

  • 代码块一定要标注语言，这样才有语法高亮
  • 代码示例要简洁，只展示关键部分
  • 复杂的代码要加注释说明
  • SQL 代码用 sql，不要用 mysql（可能不支持）

4.2 图片

提醒自己：

  • 图片放在 source/img/ 目录下
  • 引用时使用 /img/图片名.jpg
  • 替代文字要描述图片内容，方便 SEO
  • 图片不要太大，影响加载速度
  • 教程中的截图要清晰，关键部分可以用红框标注

表格对齐

提醒自己：

  • 表格适合对比多个项目的属性
  • 教程中可以用表格总结知识点
  • 表格不要太复杂，列数控制在 5 列以内

6.2 常用配置字段

字段 说明 示例
title 文章标题 title: MySQL 教程
date 发布时间 date: 2026-02-24 10:00:00

  • categories 用现有的分类，保持一致性
  • tags 可以多个，用短横线开头
  • 系列教程要统一分类名称
  • 重要文章可以设置 top 置顶

提醒自己：

  • 发布前一定要 hexo server 本地预览一下
  • 检查有没有格式问题、图片是否能正常显示
  • 部署命令是 hexo g -d（generate + deploy）

七、写作流程提醒

7.1 写作前

  • 确定文章主题和分类
  • 想好文章结构（可以用大纲）
  • 准备好需要用到的图片素材

7.2 写作中

  • 先写内容，不要纠结格式
  • 用标题划分章节
  • 代码示例要测试过再贴上去
  • 关键步骤要解释清楚

7.3 发布前检查清单

  • 标题有没有错别字
  • Front Matter 配置是否正确（分类、标签）
  • 代码块有没有标注语言
  • 图片路径是否正确，能否正常显示
  • 本地预览效果是否正常
  • 有没有明显的错别字或语病

提醒自己：

  • 不要追求完美，完成比完美重要
  • 文章可以后续修改，先发布再说
  • 读者的反馈会帮助你改进

九、常见错误与解决方法

问题 可能原因 解决方法
文章不显示 Front Matter 格式错误 检查 --- 是否成对，YAML 格式是否正确
图片不显示 路径错误 图片放在 source/img/，引用用 /img/xxx
代码没有高亮 没标注语言 在 ``` 后面加上语言名称
分类/标签不显示 拼写不一致 保持分类/标签名称一致，注意大小写
本地预览正常，部署后异常 缓存问题 执行 hexo clean && hexo g -d

十、总结

核心语法（必须掌握）

  • 标题 #
  • 段落（空行分隔）
  • 粗体 **
  • 列表 - 和 1.
  • 代码块 ```
  • 链接 []()
  • 图片 ![]()

写给自己的话

记住：

  1. 写作是一个思考的过程，不要等想清楚了再写，写本身就是思考
  2. 第一篇总是最难写的，写下去就好了
  3. 文章不需要完美，只要能帮助到别人（包括未来的自己）就有价值
  4. 持续更新比单篇质量更重要
  5. 读者的反馈是改进的动力
