八、最佳实践

提醒自己：

  1. 脚本开头加 shebang：#!/bin/bash
  2. 使用 set -euo pipefail：增强错误处理
  3. 变量用双引号包裹：避免空格和通配符问题
  4. 函数用 local 声明局部变量：避免污染全局作用域
  5. 有意义的变量名：使用大写表示常量
  6. 添加注释和文档：说明用途、参数、返回值
  7. 日志输出加时间戳：便于追踪
  8. 敏感信息用环境变量：不要硬编码密码

九、练习

  1. 编写一个脚本，检查磁盘使用率，超过 80% 时告警
  2. 编写一个脚本，批量创建用户并设置密码
  3. 编写一个脚本，分析 Apache/Nginx 日志，统计访问量和错误率
  4. 编写一个脚本，监控系统负载，超过阈值时发送邮件告警
  5. 编写一个自动化部署脚本，支持回滚功能
