MySQL 进阶教程 14-01 触发器详解

触发器是在特定事件（INSERT、UPDATE、DELETE）发生时自动执行的存储程序。本文将详细介绍触发器的概念、语法、应用场景和注意事项。

一、触发器介绍

1.1 什么是触发器？

触发器（Trigger） 是与表关联的特殊存储过程，当表中发生 INSERT、UPDATE 或 DELETE 事件时自动执行。

1.2 触发器作用

作用 说明
数据验证 检查数据有效性
自动计算 自动更新相关字段
审计追踪 记录数据变更历史
级联操作 自动维护相关表

1.3 触发器限制

  • 每个表每个事件每种时机只能有一个触发器
  • 触发器不能调用存储过程（某些版本限制）
  • 触发器不能有输入输出参数
  • 触发器不能使用 COMMIT/ROLLBACK

2.2 触发器时机

时机 说明
BEFORE 在操作执行之前触发
AFTER 在操作执行之后触发

2.3 NEW 和 OLD 关键字

关键字 INSERT UPDATE DELETE
NEW 新插入的值 新值 无
OLD 无 旧值 删除的值

7.3 触发器最佳实践

  1. 保持触发器简短：避免复杂逻辑
  2. 避免递归触发：可能导致无限循环
  3. 注意性能影响：触发器会增加操作开销
  4. 明确文档化：记录触发器的作用和逻辑
  5. 谨慎使用：能用外键解决的不用触发器

八、小结

核心知识点

  1. 触发器类型：BEFORE/AFTER + INSERT/UPDATE/DELETE
  2. NEW 和 OLD：访问新值和旧值
  3. 应用场景：审计、同步、约束、级联操作
  4. 注意事项：保持简短、避免递归、注意性能

触发器对比

时机 适用场景

AFTER INSERT 日志记录、级联插入
BEFORE UPDATE 数据验证、计算字段
AFTER UPDATE 日志记录、数据同步
AFTER DELETE 日志记录、级联删除
