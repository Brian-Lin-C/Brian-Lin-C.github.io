MySQL 进阶教程 04-01 窗口函数概述

窗口函数（Window Function）是 MySQL 8.0+ 引入的强大特性，用于实现复杂的分析查询。本文将详细介绍窗口函数的基本概念、语法结构以及与聚合函数的区别。

一、窗口函数简介

1.1 什么是窗口函数？

窗口函数 是对一组行（称为”窗口”）执行计算的函数，但与聚合函数不同的是，窗口函数会为每一行返回一个结果，而不是将多行聚合为单个值。

通俗理解：

  • 聚合函数：多行 → 一个结果
  • 窗口函数：多行 → 每行一个结果（基于窗口内数据计算）

1.2 窗口函数的应用场景

场景 说明 示例
排名问题 获取 Top N 记录 每个部门工资最高的 3 名员工
累计计算 累计求和、平均值 累计销售额、移动平均
前后行访问 访问当前行的前后行数据 同比、环比增长
分组内排序 组内排序编号 每笔订单在客户订单中的序号
分桶分析 将数据分成若干组 按成绩分等级、百分位分析

1.3 为什么需要窗口函数？

在窗口函数出现之前，实现类似功能通常需要：

  • 自连接
  • 子查询
  • 用户变量
  • 应用层处理

这些方法往往语法复杂、性能较差、可读性低。窗口函数提供了更简洁、高效、易读的解决方案。

3.1 基本语法

3.2 语法组件详解

（1）OVER 子句

OVER 子句 是窗口函数的标志，没有 OVER 就不是窗口函数。

（2）PARTITION BY 子句

PARTITION BY 将数据划分为多个分区（窗口），函数在每个分区内独立计算。

PARTITION BY vs GROUP BY：

  • GROUP BY：分组后，每组返回一行
  • PARTITION BY：分区后，每行保留，附加计算结果

（3）ORDER BY 子句

ORDER BY 在窗口内对数据进行排序，影响排名和累计类函数。

（4）窗口边界子句

ROWS/RANGE BETWEEN 定义窗口的范围（用于累计、移动平均等）。

窗口边界详细说明见后续章节。

四、窗口函数分类

MySQL 窗口函数主要分为以下几类：

4.1 排名函数

函数 说明
ROW_NUMBER() 连续排名，无重复，不跳过（1,2,3,4…）
RANK() 跳跃排名，有重复，跳过（1,2,2,4…）
DENSE_RANK() 密集排名，有重复，不跳过（1,2,2,3…）
NTILE(n) 将数据分成 n 个桶

4.2 聚合函数（窗口版）

MySQL 的所有聚合函数都可以用作窗口函数：

函数 说明
SUM() OVER 窗口求和
AVG() OVER 窗口平均
COUNT() OVER 窗口计数
MAX() OVER 窗口最大值
MIN() OVER 窗口最小值

4.3 分析函数

函数 说明
LEAD(col, n) 访问当前行之后第 n 行的值
LAG(col, n) 访问当前行之前第 n 行的值
FIRST_VALUE(col) 窗口内第一行的值
LAST_VALUE(col) 窗口内最后一行的值
NTH_VALUE(col, n) 窗口内第 n 行的值

六、窗口函数的执行顺序

理解窗口函数的执行顺序对于编写正确的 SQL 至关重要。

关键点：

  • 窗口函数在 WHERE、GROUP BY、HAVING 之后执行
  • 窗口函数在最终 ORDER BY 之前执行
  • 可以在外层查询中对窗口函数的结果进行过滤

八、小结

核心知识点

  1. 窗口函数特点

    • 为每一行返回一个结果
    • 不改变原始数据
    • 基于窗口（一组行）进行计算

  2. 与聚合函数的区别

    • 聚合函数：多行 → 一个结果
    • 窗口函数：多行 → 每行一个结果

  3. OVER 子句组件

    • PARTITION BY：划分分区
    • ORDER BY：窗口内排序
    • ROWS/RANGE BETWEEN：窗口边界

  4. 窗口函数分类

    • 排名函数：ROW_NUMBER、RANK、DENSE_RANK、NTILE
    • 聚合函数：SUM、AVG、COUNT、MAX、MIN
    • 分析函数：LEAD、LAG、FIRST_VALUE、LAST_VALUE
